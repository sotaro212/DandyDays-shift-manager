# シフト管理Webアプリ 仕様書

> バージョン: 1.0  
> 作成日: 2026-05-03  
> 想定利用: Dandy Days（初回）／他店舗への転用を想定した汎用設計

---

## 1. サービス概要

キッチンカー事業者向けのシフト管理Webアプリ。  
管理者がシフト枠（場所・人数）を登録してURLを発行し、アルバイトスタッフがブラウザから希望を回答する。回答順に自動充当し、管理者が最終調整して確定する。

### 想定規模（Dandy Days）
| 項目 | 内容 |
|------|------|
| 管理者 | 5名 |
| アルバイト | 約30名 |
| キッチンカー | 最大1日4台 |
| 月間稼働日数 | 約25日 |

### 他店舗転用時の変更箇所
- 管理者数・バイト数（スケール変更のみ）
- 月間稼働日数（リマインドメール通数の試算に影響）
- 場所マスタ（登録する場所名）

---

## 2. 利用者と操作

| 利用者 | アクセス手段 | 主な操作 |
|--------|-------------|---------|
| 管理者（オーナー権限） | PC / スマホブラウザ | シフト作成・確定・スタッフ管理・全権限 |
| 管理者（編集権限） | PC / スマホブラウザ | シフト作成・確定・スタッフ管理 |
| アルバイト | スマホ / PCブラウザ（URL共有） | シフト希望の回答・確認 |

### 管理者権限ルール
- 各キッチンカーに **管理者が最低1名以上アサインされていること**（シフト確定時にバリデーション）
- 自身の権限は変更不可（誤操作防止）
- 管理者が1名のみの状態での降格は不可

---

## 3. 基本フロー

```
① 管理者：場所・必要人数を日付ごとに登録
         ↓
② 管理者：その月のシフト表URLを発行（締め切り日時を設定）
         ↓
③ バイト：URLにアクセスして希望枠を回答（ブラウザのみ・ログイン不要）
         ↓
④ システム：回答順に自動でシフト枠へ充当（先着順）
         ↓
⑤ 管理者：残りを自由にカスタマイズ・調整して確定
         ↓
⑥ システム：シフト前日にバイトへリマインドメールを自動送信
```

## 4. 機能要件

### 4.1 シフト管理

#### シフト枠の登録
- 1シフト枠 = **日付 × 場所 × 必要人数**（時間帯は管理しない）
- 月単位でまとめて登録可能
- 既存月のシフト枠を別の月にコピー可能（日付維持 / 曜日維持を選択）
- 特定日のシフト枠を複数日にコピー可能
- 各シフト枠を個別に編集・削除可能

#### URLの発行と締め切り
- 月ごとにシフト回答用URLを1本発行
- 締め切り日時を任意で設定
- 管理者が手動で即時締め切り可能
- 締め切り後はバイトのアクセスを遮断し「募集終了」を表示

#### シフト充当と確定
- バイトの回答が届き次第、**先着順**でシフト枠に充当
- 充当状況はリアルタイムで管理者に表示
- 管理者はアサイン済みスタッフの追加・差し替え・削除が可能
- 未充足の枠は赤色バッジで強調表示
- シフト確定時に「各キッチンカーに管理者1名以上」をバリデーション

#### シフトステータス
| ステータス | 条件 |
|-----------|------|
| draft（案） | 作成直後・調整中 |
| confirmed（確定） | 管理者が確定操作済み |
| undecided（未確定） | 必要人数に満たない |

### 4.2 バイト向け回答機能
- URLにアクセスするだけで回答可能（会員登録・ログイン不要）
- 初回アクセス時に名前・メールアドレス・住所（市区町村）を入力して登録
- 希望する日付・場所を選択して送信
- 回答済みの枠は再アクセス時に編集可能（締め切り前まで）
- 締め切り後はすべて閲覧のみ

### 4.3 スタッフ管理（管理者）
- スタッフ一覧の表示（名前・メール・住所・権限・登録日・最終アクセス）
- 権限変更（user ↔ admin）
- スタッフの削除（退職者整理）
- 変更操作はログに記録

### 4.4 近隣場所の割り振り補助
- 同じ日に複数の場所がある場合、バイトの居住市区町村から近い場所を**提案表示**
- 最終的なアサインは管理者が決定（自動確定はしない）

### 4.5 リマインド通知
- 送信対象: **アルバイトのみ**（管理者へは送らない）
- タイミング: シフト確定日の前日
- 内容: 「【明日のシフト】〇月〇日 @ 場所名」
- 未確定（undecided）のスタッフには送らない

## 5. 非機能要件

| 項目 | 内容 |
|------|------|
| 対応デバイス | PC・スマホ両対応（レスポンシブデザイン） |
| 認証（管理者） | Googleアカウント OAuth |
| 認証（バイト） | URLトークン認証（ログイン不要） |
| データ保持 | 管理者自身のGoogle Sheetsに保存 |
| セキュリティ | バイトはGASエンドポイント経由でのみ書き込み（シート直接アクセス不可） |
| 運用コスト | **永続0円** |

## 6. データ設計（Google Sheets 構成）

### シートA: members（スタッフ情報）
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| name | String | 氏名 |
| email | String | メールアドレス（リマインド送信先） |
| city | String | 居住市区町村 |
| role | user / admin | 権限 |
| created_at | Timestamp | 登録日時 |
| last_accessed_at | Timestamp | 最終アクセス |

### シートB: shift_months（月別シフト管理）
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー（URLに使用） |
| year | Int | 対象年 |
| month | Int | 対象月（1〜12） |
| status | draft / published / closed | 募集ステータス |
| deadline_at | Timestamp | 締め切り日時（任意設定） |
| published_at | Timestamp | 公開日時 |
| closed_at | Timestamp | 締め切り実施日時 |

### シートC: shift_slots（シフト枠）
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| shift_month_id | UUID | shift_months への参照 |
| location_name | String | 場所名 |
| date | Date | シフト日付 |
| required_count | Int | 必要人数 |
| status | draft / confirmed / undecided | 確定ステータス |
| note | Text | 備考 |

### シートD: staff_responses（回答・アサイン）
| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | 主キー |
| shift_slot_id | UUID | shift_slots への参照 |
| member_id | UUID | members への参照 |
| is_available | Boolean | 希望あり/なし |
| submitted_at | Timestamp | 回答日時（先着順判定に使用） |
| is_assigned | Boolean | 確定アサインフラグ |

## 7. アーキテクチャ

```
[管理者ブラウザ]
     │ Google OAuth
     ▼
[GitHub Pages] ──── Google Sheets API ────▶ [管理者のGoogle Sheets]
     │                                              ▲
     │ URLトークン発行                              │ GASエンドポイント経由
     ▼                                              │（直接アクセス不可）
[バイトブラウザ] ──────────────────────────────────┘

[リマインド]
Google Apps Script タイマー → EmailJS / Gmail → バイトのメールアドレス
```

### 使用サービスとコスト
| サービス | 用途 | コスト |
|---------|------|--------|
| GitHub Pages | フロントエンドホスティング | 永続0円 |
| Google Sheets | データベース（管理者所有） | 0円 |
| Google OAuth | 管理者認証 | 0円 |
| Google Apps Script | バイト回答の書き込みエンドポイント / リマインドバッチ | 0円 |
| EmailJS（無料枠） | リマインドメール送信（200通/月） | 0円 |

### リマインドメール通数試算（Dandy Days）
- 通常運用: 25日 × 平均5名 = **125通/月**（無料枠内）
- 最大想定: 25日 × 12名 = **300通/月**（超過）→ GAS + Gmail送信に切り替えで0円維持

### Google Sheets アクセス権限
| 対象 | 権限 | 方法 |
|------|------|------|
| オーナー管理者（1名） | フル権限 | Googleアカウントでシート所有 |
| 共同管理者（1名） | 編集権限 | Googleアカウントで共有 |
| アルバイト（約30名） | 書き込みのみ（間接） | GASエンドポイント経由 |

## 8. エラー・例外挙動

| ケース | 挙動 |
|--------|------|
| 締め切り後にURLアクセス | 「このシフトの募集は終了しました」を表示 |
| 未登録スタッフのアクセス | 登録フォームへ遷移 |
| シフト確定時に管理者アサインなし | 「各キッチンカーに管理者が必要です」エラーを表示 |
| 管理者が自身の権限を変更しようとした | 「自身の権限は変更できません」を表示 |
| 最後の管理者を降格しようとした | 「管理者が1名のため降格できません」を表示 |
| シフトコピー時に重複枠が存在 | 確認モーダルで「スキップ／上書き」を選択 |
| 回答済み枠への再アクセス（締め切り前） | 過去の回答を表示した状態で編集可能 |

## 9. 将来対応・拡張メモ

- **他店舗転用**: 場所マスタと管理者アカウントを変えるだけで別店舗として独立運用可能
- **PWA化**: GitHub Pages + next-pwa でホーム画面追加・オフライン対応が容易
- **通知手段の拡張**: LINEへの切り替えも可能（LINE Messaging API、月200通無料枠）
- **時間帯管理の追加**: 必要になった場合は shift_slots に start_time / end_time を追加
- **多言語対応**: バイトが外国籍の場合は i18n 対応を検討
