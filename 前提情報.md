# Dandy Days シフト管理アプリ 前提情報

## 会社情報
- **会社名**: Dandy Days

## 人員構成
| 区分 | 人数 |
|------|------|
| 管理者権限スタッフ | 5名 |
| アルバイト | 約30名 |
| 合計 | 約35名 |

## キッチンカー運用
- 最大稼働台数: **1日4台**
- 月間稼働日数: **約25日**
- 各キッチンカーには **管理者権限を持つ人が最低1名以上** 必要（必須ルール・バリデーション対象）
- どのキッチンカーを稼働させるかは管理者が判断（システム管理不要）

## シフト仕様
- 管理項目: **場所と人数のみ**（時間帯は管理しない → start_time / end_time は不要・削除）
- 1シフト枠 = 「日付 × 場所 × 必要人数」

## 基本フロー
1. 管理者が場所と人数を登録し、その月のシフト表URLを発行
2. URLには **締め切り日時を自由に設定**でき、管理者が任意のタイミングで手動締め切りも可能
3. バイトはURLにアクセスして希望を回答
4. 回答が来た順に自動的にシフト枠へ充当（先着順）
5. 管理者が残りを自由にカスタマイズして確定

## スタッフ情報
- バイトの**住所（市区町村レベル）を登録**する
- 同じ日に複数の場所がある場合、**居住地に近い場所**へ優先的に割り振る補助機能を提供
- 最終的なアサインは管理者が判断（自動割り振りはあくまで提案）

## リマインド通知
- 送信対象: **アルバイトのみ**（管理者には送らない）
- 手段: メール（EmailJS 無料枠）
- タイミング: シフト前日
- 月間通数試算:
  - 稼働25日 × 平均アサインバイト数（約4〜6名/日）= **100〜150通/月**
  - EmailJS 無料枠（200通/月）**以内で収まる見込み**
  - ※ 全4台フル稼働・全枠バイトのみで最大試算: 25日 × 12名 = 300通 → 超過リスクあり
  - 超過時の代替: Google Apps Script（Gmail送信）に切り替え → **0円のまま維持可能**

## アーキテクチャ（確定）
| 要素 | サービス | コスト |
|------|---------|--------|
| ホスティング | GitHub Pages | 永続0円 |
| DB | 管理者のGoogle Sheets | 0円 |
| 管理者認証 | Google OAuth | 0円 |
| スタッフ回答 | GASエンドポイント経由で書き込み | 0円 |
| リマインドメール | EmailJS（超過時はGAS）| 0円 |
| ドメイン | github.io サブドメイン | 0円 |

## Google Sheetsアクセス権限
- **シートオーナー（1名）**: フル権限
- **共同編集者（1名）**: 編集権限付与 → 合計2名が直接シートを操作可能
- **スタッフ（約30名）**: 直接アクセス不可。Google Apps Scriptのエンドポイント経由でのみ書き込み
  - セキュリティ上、スタッフはシートのURLや構造を見えない状態で回答できる

## 補足事項
- 管理者5名はキッチンカーへのアサイン対象でもある
- キッチンカーは最大4台なので、管理者5名で各台をカバーできる
- LINEは不使用（URLをLINE/SMS/メール等で共有するだけ）

| 項目 | 仕様書の記載 | 実際の仕様 |
|------|-------------|-----------|

| 管理者アサインバリデーション | 未記載 | 追加 |
| 住所（市区町村）管理 | 未記載 | members に追加 |

| 通知手段 | LINE Messaging API | メール（EmailJS / GAS） |
| 通知対象 | 全確定スタッフ | アルバイトのみ |
| インフラ | Vercel + Supabase | GitHub Pages + Google Sheets |
